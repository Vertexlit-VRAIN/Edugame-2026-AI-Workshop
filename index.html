// Demo 01

System 2 Prompting

Activating deliberative reasoning — specificity determines whether the model thinks or just reacts

What you show
Two prompts, same question. The vague version triggers a reflexive, automatic response — System 1. The structured version, with explicit role, context, and chain-of-thought instruction, forces the model to pause, generate reasoning paths, and evaluate before answering — System 2.
Why it lands
The output difference is visible in seconds. Faculty recognise it immediately as analogous to the difference between an ambiguous assignment brief and a well-scoped one. The audience just learned that AI can reach System 2 — this shows the switch that turns it on.
Key Insight
AI no longer defaults to its first automatic prediction — but only if you give it a reason not to. That reason is the prompt. Prompt design is a teachable pedagogical skill, not a technical one.
// Demo 02

Claude Cowork

The Agent Action Loop made visible — observe, generate sub-programs, execute, evaluate

What you show
The BIP AI Workshop — a complete, offline interactive educational tool built in a single Cowork session. The agent observed the task, generated intermediate file structures, executed across the filesystem using its full toolkit, and evaluated its own output. No code written by hand. This presentation is also a product of that same loop.
Why it lands
The audience sees a real artefact produced by an agent cycling through the exact loop described in the theory. The agent toolkit — file access, code execution, web browser — was used autonomously to produce something any educator could deploy tomorrow. It answers "but can it do something I would actually use?"
Key Insight
The agent holds the toolkit. The faculty holds the goal. The constraint on what an educator can produce is no longer technical — it is entirely the quality of the goal specification.
// Demo 03

OpenClaw

The Delegation Architecture in practice — one instruction, distributed execution, no human bottleneck

What you show
From a phone, one Telegram message reaches a local OpenClaw instance. The Main Agent receives the goal, decomposes it into sub-activities — search, process, synthesize — coordinates their execution, and delivers a finished file to the Desktop. No further human input at any stage.
Why it lands
This is the Fractal Intelligence Framework running on a laptop, deployed in an afternoon. The theory described parallel specialized agents operating without a human bottleneck — this is what that looks like as a working system, open-source, locally controlled, and institutionally deployable by any faculty member.
Key Insight
The barrier to a Delegation Architecture on institutional infrastructure is now very low. That is both the opportunity and the governance question that cannot stay theoretical.
// Implications & Discussion

What Changes — What to Discuss

01 — Specify
Define the goal clearly enough that an agent can pursue it. This is a design skill that can be taught.
02 — Supervise
Intervene with domain knowledge the agent cannot replicate. Identify when it is confident and wrong.
03 — Review
Evaluate output against standards the agent does not hold. Faculty becomes quality assurance, not execution.
