What Is an Agent?

Chatbot — one turn at a time
  • 01 You write a prompt
  • 02 The model generates text
  • 03 You read the output
  • 04 You decide what to do next
  • 05 You write the next prompt
  • — Repeat until done
Agent — goal to result
  • 01 You define a goal
  • 02 The agent plans the steps
  • 03 The agent uses tools — search, code, files
  • 04 The agent monitors its own progress
  • 05 The agent delivers the result
  • — You review

Prompt Design

The human in full control — goal quality determines output quality

What you show
A structured multi-role prompt given to Claude in a single turn. Assign a persona, a context, and a compound task — then compare the output to a vague version of the same question side by side.
Why it lands
The quality difference is immediately visible to any educator. It also shows the boundary: where specificity helps, and where domain expertise is still the only answer.
Key Insight
Writing the prompt IS the design decision. That is a teachable skill, not a technical one.
// Demo 02

Claude Cowork

The agent in the production pipeline — you supervise, not execute

What you show
The BIP AI Workshop "Prompt Wars" — a complete, offline interactive workshop built with Cowork. Three structured parts, custom visual design, no code written by hand.
Why it lands
It answers "but can it do something I'd actually use?" with a real artefact the audience can see, navigate, and immediately recognise as production-ready educational content.
Key Insight
The constraint on what an educator can produce is no longer technical. It is now entirely the quality of the goal specification.
// Demo 03

OpenClaw

The human at the perimeter — always-on, always reachable

What you show
From a phone, send one Telegram message to a running OpenClaw instance. It searches, processes, and delivers a finished file to the Desktop — without any further input.
Why it lands
An open-source agent, running on a laptop, deployed in an afternoon, reachable from any device. It demonstrates the logical endpoint of the agentic direction — and forces the governance question.
Key Insight
The barrier to autonomous AI acting on institutional infrastructure is now very low. That is both an opportunity and a policy question that needs to be on the faculty agenda.
// Implications

What Changes for Faculty

01
Specify
Define the goal clearly enough that an agent can pursue it without constant correction. This is a design skill that needs to be taught explicitly.
02
Supervise
Monitor execution. Identify when the agent is confident and wrong. Intervene with the domain knowledge the agent cannot replicate.
03
Review
Evaluate output against standards the agent does not hold. The faculty role becomes quality assurance, not step-by-step execution.
